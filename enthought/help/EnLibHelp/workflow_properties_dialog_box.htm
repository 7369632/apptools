• Changing Workflows

 

Workflow Properties Dialog Box

To access this dialog box: Double-click a workflow or case node under Workflows or Cases in the ProAVA2 Explorer tab.

Use this dialog box to change settings for workflows (a case is a type of workflow).

CAUTION: It is possible to construct a workflow that cannot be executed. An invalid workflow causes an error message when you try to open a Synthetic View or Simulation View window. It is a good idea to make a back-up copy of your project before changing a valid workflow.

Name: The name of the workflow.

Components: On the left side of the dialog box is a list of components that implement the workflow. Select a component name to display its parameters on the right side of the dialog box.

Inputs: The fields in this area vary, depending on the component selected. The field names are parameter names, and the values are literal values or variable names.

Outputs: The field in this area varies, depending on the component selected. The field name is a description of the output, and the field value is a variable name, which can be used in the input fields of other functions.

Buttons for components:

Button

Description

Displays the Component Library Browser dialog box, so that you can select a component to add to the workflow.

Deletes the selected component from the workflow.

Moves the selected component up in the function list.

Moves the selected component down in the function list.

Enables the selected component , if it is disabled.

Disables the selected component, if it is enabled.
