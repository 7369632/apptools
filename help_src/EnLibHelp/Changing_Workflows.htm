Changing a Workflow

You can change the following aspects of a workflow or case:

  • Parameter values for functions

  • Add or remove functions in the workflow

  • Enable or disable functions in the workflow

  • Change the order of functions in the workflow

CAUTION: It is possible to construct a workflow that cannot be executed. An invalid workflow causes an error message when you try to open a Synthetic View or Simulation View window. It is a good idea to make a back-up copy of your project before changing a valid workflow.

To change a workflow:

  1. Double-click a workflow or case node in the ProAVA2 Explorer tab. The Workflow Properties dialog box appears.

    • To change the inputs to a function, select the function on the left side of the dialog box, and edit the input values on the right side of the dialog box.

    • To add a function, click , and then select a function in the Component Library Browser dialog box.

    • To remove a function, select the function on the left side of the dialog box, and click .

    • To change the order of functions, click to move the selected function up, and click to move the selected function down. The functions execute in the order that they are listed.

    • To disable a function, select the function and click . The word "(disabled)" appears next to the function name in the function list. Disabling prevents the function from executing, without removing it from the workflow definition.

    • To enable a disabled function, select it and click . The word "(disabled)" disappears from next to the function name.

  2. Click OK when you are finished making changes.
